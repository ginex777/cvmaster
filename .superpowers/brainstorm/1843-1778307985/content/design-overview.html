Landing Page — Angular Implementation Design

Pixel-perfect replication of the standalone HTML. Four concerns to resolve before building.

1 · Design token sync styles.css must be updated first

The standalone HTML uses a different token set than the current styles.css. Every colour, radius and shadow value differs. For a 100% match, styles.css must be replaced before any component work starts.

Current styles.css (wrong)
--bg: oklch(98.6% 0.003 247) ← cool/blue
--radius-sm: 4px
--radius-md: 8px
--shadow-sm: 0 1px 3px oklch(20% 0 0/10%)
missing: --surface, --line, --grain
missing: --accent-ink, --shadow-glow
missing: --max, --pad
Standalone HTML (correct)
--bg: oklch(98.6% 0.005 80) ← warm/beige
--radius-sm: 8px
--radius: 14px (new mid token)
--shadow-sm: 0 1px 2px rgba(15,18,32,.04)…
--surface, --surface-2, --line, --line-2
--accent-ink, --shadow-glow, --grain
--max: 1200px · --pad: clamp(20px,4vw,40px)

2 · Button component — variants to update

The standalone HTML uses 4 button variants. The current component has primary/secondary/ghost/danger — needs to become primary/ghost/outline/accent.

primary (dark) ghost outline accent (blue)
